# 校验规则

本规则用于 `validate_structured_data.py` 和人工复核，目标是让 Core 只继承可追溯的数据。

## 必须存在的章节

- 元信息、数据完整性、太极与命身、七政四余星曜位置。
- 十二宫、二十八宿、神煞、原局初步标记、运势资料、校验结果。

## 字段完整性

- 命宫、命度、命度主、身宫、身度、身度主不能全空。
- 七政四余必须至少有 11 行：日、月、金、木、水、火、土、罗、气、计、孛。
- 每张表都应保留来源和可信度；没有来源时不得标高可信度。
- 性别缺失时，男命/女命取法必须列为待确认项。

## 一致性检查

- 命度、身度、星宿/度是否与二十八宿表一致。
- 十二宫内星曜是否能在星曜位置表中找到。
- 神煞是否标明位置和作用领域，不能只列名称。
- 流年、限度、倒限要与原局字段分开，避免混读。

## 进入 Core 的条件

关键命身字段缺失时，只能生成补数清单；字段有低可信度时，Core 只能做初步框架并标注不确定项。
