# 缺失数据处理规则

Reader 的职责不是“猜出来”，而是把缺口暴露清楚，交给用户或上游资料补齐。

## 必须暂停的缺失

- 缺命宫、命度、命度主、身宫、身度、身度主：不得进入 Core 正式分析。
- 缺性别：不能应用男命/女命取法，只能写“取法待确认”。
- 七政四余缺任一星曜：不得做五行偏性、尽三或余奴相犯判断。

## 可以继续但要降级的缺失

- 缺神煞：可读原局结构，但神煞相关专题降级。
- 缺流年/限度：可分析原局，不做应期和运势引动。
- OCR 可信度低：输出初稿，并要求用户核对盘面原图或文本。

## 待确认项写法

- 写清字段名、当前识别值、可疑原因、建议补充方式。
- 不使用“应为”“必定是”等推断式措辞。
- 示例：“命度主识别为金，来源 OCR，可信度低；请用户核对原盘命度所在星宿及度主。”
